{% extends "_post.html" %} {%hyde title: Gmailの二段階認証プロセス(ワンタイムパスワード)概要 created: 2012-12-27 04:41:00 %} {% block article %} {%article%} {% excerpt %} 　　 - [Gmail乗っ取りが大流行中！被害報告まとめ](http://togetter.com/li/428959) 　Gmailがワンタイムパスワード(※)に対応していることを知らない人もいるようなので、手間や使用可能な環境など設定する前に疑問となるような点についてまとめます。 設定については手順通りに進めればわかりづらいところは無いと思います。 　
　 *※Googleの呼称は「二段階認証プロセス」と「確認コード」* 　
　 ##どこから設定するのか - [アカウント設定](https://www.google.com/settings/security) 　
　 ##ワンタイムパスワードはどうやって生成するのか **携帯端末(Android, iOS, BlackBerry)**にインストールした**アプリ**で行う。 - [Google認証システム - Google Play の Android アプリ](https://play.google.com/store/apps/details?id=com.google.android.apps.authenticator2) 　
　 ##ワンタイムパスワードを生成する携帯端末の認証方法 最初にワンタイムパスワードを生成する端末の認証を行う必要がある。 大多数の人は**SMS**で携帯に送られるコードを入力して端末の認証を行うと思われるが、**電話番号**に**音声でコードを送る**方法もあるので、**Nexsus7**と**Skype番号**しか持っていないような環境でも問題ない。 　
　 ##WindowsのOutlookやMacのMailなどはどうなるのか 　ワンタイムパスワードに対応していないので、一度だけ表示される12桁の**アプリケーション固有のパスワード**を生成し、それをパスワードとして設定する。 ![account_infomation](/media/blog-images/account_infomation.jpg) 固有のパスワードを生成する必要のあるアプリケーションは以下のようなものがある。 >- POP および IMAP メール クライアント（Outlook、Mail、Thunderbird など） >- スマートフォンの Gmail、Google カレンダー >- Windows Mobile、iPhone 用の ActiveSync >- Apple 端末での YouTube モバイル >- クラウド プリント >- インストールされているチャット クライアント（Google トーク、Adium など） >- 3D ギャラリー、SketchUp、インストールされているアプリケーション >- AdWords Editor >- Google Chrome の同期機能 >- Gmail Notifier - [アプリケーション固有のパスワードを使用してログインする - Google アカウント ヘルプ](http://support.google.com/accounts/bin/answer.py?hl=ja&answer=185833) 　
　 ##自宅のコンピュータのブラウザでログインする場合、毎回ワンタイムパスワードを入れなければならないのか 　**信頼できるパソコン**として登録することで、ワンタイムパスワードの入力を省ける。信頼できるパソコンリストから除外することで手間はかかるが、よりセキュリティを向上させることも可能。 　
　 ##Google認証システムアプリをいれた端末を紛失したら 　認証システムアプリをインストールした端末を紛失したり、家に忘れてきて外出先でGmailにログインしたい時、どのようなバックアップシステムがあるのか。 - 第一に、**バックアップ用の電話番号**を設定しておくことができる。 　ここに送られる確認コードを入力することでログインができる。 - 第二に、**印刷用バックアップコード**を印刷しておくことができる。 　携帯を家に忘れてきた場合、設定しておいたバックアップ用の電話番号も同時に使えないことがほとんどだろう。二重のバックアップとして、あらかじめ8桁のバックアップ確認コードを10個生成しておくことができる。これを印刷して財布などに入れておけばよい。 ちなみにこの確認コードは一度使うと二度と使うことができなくなる。つまり生成した10個全ての確認コードを使用してしまった場合は、新たに10個のバックアップ確認コードを生成する必要がある。 {%endexcerpt%} {%endarticle%} {% endblock %}
